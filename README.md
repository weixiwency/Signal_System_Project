# Signal_System_Project

## 人员1
把原始的含噪音频变成了可直接训练的纯净数据。具体核心产出为以下四点：
1. 基础建设： 搞定了 Python 环境依赖，搭建了 GitHub 协作仓库并完成了团队授权。
2. 可视化与报告： 跑出项目要求的 4 张核心图表（波形图、频谱图、语谱图、MFCC），并完成它们对应的物理原理解析。
3. 数据清洗： 优化端点检测（VAD）算法，利用“时长过滤”和“能量排行”机制，剔除录音里的杂音和呼吸声，截取 0~9 的发音片段。
4. **数据集增强** ：在初期实验中，我们发现无噪声验证集的识别准确率存在瓶颈（约 70%），且模型对特定测试者（如 Speaker 6）的特定发音极易产生误判。经过分析，这是由于本实验训练样本极度匮乏（仅 50 个），模型陷入了小样本过拟合现象，泛化能力不足。为此，我们在特征提取端（预处理阶段）引入了音频数据增强策略。针对切分出的有效发音片段，我们分别进行了：音高平移（Pitch Shifting）：上下平移 2 个半音，模拟不同性别和声带结构的发音。白噪声注入（Noise Injection）：以 0.5% 的峰值振幅混入高斯白噪声。通过该策略，我们将训练集无损扩充了 4 倍（由 50 增至 200 样本）。重新训练后，SVM 与 MLP 模型的基线准确率获得了大幅跃升，彻底解决了因泛化能力弱导致特定发音无法识别的难题。
5. 交付： 将变长的物理声音降维成了 13 维定长的数学特征，打包生成了纯净、零错位的 speech_features.npz 供下一个负责模型的同学直接调用。
